# 收录判据 + 板块锚点 + 文风

## 四道收录闸（替代"模块名+关键词"硬匹配）
一条候选要进，须同时满足：
1. **主题门槛（由节点①确定，主题可变）**：默认 = 〔AI/数智：生成式AI、代理式AI、算法、自动化、ML/NLP、机器人〕×〔工作/组织/人：工作设计、组织、管理、HR各模块、技能、劳动力市场、员工体验/心理〕。若用户本期指定其他 OBHRM 主题（组织变革/文化/招聘甄选/薪酬/绩效/员工发展/SSC/领导力/团队动力学…），则门槛 = 〔该主题〕×OBHRM，且**须能归入本期用户所给板块之一**。纯无关一律排除。
2. **证据门槛**：优先实证/理论建构/系统综述；工作论文须作者+拟发表刊物可信。
3. **质量门槛**：见 source_registry（A类 DOI+索引；A2 看机构+forthcoming+DOI；新来源→候选隔离区）。
4. **时效门槛**：发表/更新日期落在本期检索窗口内。

## 精判方式
粗筛（白名单+窗口+主题门槛）出候选池后，**逐条读标题+摘要**判断：是否在范围内、归哪个板块、与历期是否重复、置信度、一句入选理由。**低置信度边缘条目单列给用户定夺**（保召回 + 人审兜底）。

## 五大板块 + 锚点样例（few-shot）
**§1 技术的发展对工作的影响** —— AI 对任务/技能/工资/知识工作的微观与实证影响。
例：生成式AI重构知识工作范式；AI赋能工作非常规化对创造力的影响；AI、工作任务、技能与工资；AI与技术性失业；财会领域人机协同与生产力。

**§2 技术的发展对组织的影响** —— AI 对团队/决策/组织结构/治理与理论的影响。
例：AI伪装CEO与算法厌恶悖论；生成式AI对团队决策质量的影响；在组织与管理中理论化AI；智能体组织的异步推理；基于积极组织学的AI公平设计。

**§3 人的相关技能** —— 技能、人机协作能力、职业自我、学习与适应。
例：未来工作自我显著性综述；员工与生成式AI协作的暗面；人机伙伴关系的演变与管理；跨越认知不对称的人机协作。

**§4 安全、隐私、员工感受、社会热点事件** —— 伦理/隐私/公平/员工心理 + 时事热点（此板块最常出现公众号社媒型条目）。
例：信任崩塌"玻璃心"员工；AI替代工作的补偿性消费；AI技术性失业的自尊问题；算法管理与零工安全；包容性AI与招聘偏见。

**§5 公司的最佳实践** —— 具体企业案例与落地做法，多来自 B类/公众号，标题更"标题党"。
例：跨国公司薪酬暗战；中层是培训转化的隐形引擎；微软软件工程人机协同四种模式；百度职级体系大调整。

## 体量基线（灵活）
每期总条目 17–30，均值≈23。各板块经验配额：§1≈4–5；§2≈4–8(最常扩容)；§3≈3–4；§4≈3–9；§5≈2–3。

## 两种摘要文风
- **学术型**（§1–§4 多数）：背景/缺口 → 理论 → 方法/数据 → 发现，含具体统计量（p值、效应量、百分比）。约 200–400 字。
- **社媒/案例型**（部分§1、§4，多数§5）：标题更"标题党"（问号/冒号/引号），叙事式摘要，结尾给启示或追问。

## 每条结构
加粗中文标题 → 中文摘要 → `【原文信息】：` 引文（期刊全引 或 链接）。

## 摘要生成逻辑 + 主旨保真（可审）
**目的**：确保每条摘要确实反映原文主旨，不臆造、不跑偏。

**取材规则（保真核心）**：
1. 摘要**只能基于实际取到的原文文本**——优先全文，其次官方摘要/作者摘要；**取不到全文时只据摘要写，并在内部标记"仅据摘要"**，不替作者补充未见内容。
2. **不臆造任何数字**（p值、效应量、样本量、百分比）：只有在原文出现时才写；未见到就用定性表述（"显著正向""作者报告改善"），绝不编造具体数值。
3. 学术型严格按 **背景/缺口 → 理论 → 方法/数据 → 发现** 复述作者自己的主张；案例/社媒型按 **现象 → 做法/机制 → 启示** 复述，且**回溯到原始论文**后以原文主张为准（社媒解读与原文冲突时以原文为准）。
4. 标题须由摘要内容支撑，不得为"标题党"而偏离主旨。

**逐条自检（写完每条后过一遍）**：
- 这条摘要的每个论点，是否都能在我取到的原文文本里找到出处？
- 有没有出现原文里没有的具体数字？
- 主旨句（这篇到底说了什么）是否就是作者的核心结论，而非我的引申？
- 若仅据摘要/取不到全文，是否已标注，提示用户复核？

**给用户的可审产物**：节点⑤评审时，每条给"一句话主旨 + 一句入选理由 + 置信度 + 是否仅据摘要"，用户可逐条核对逻辑与保真度后再决定写长摘要。
