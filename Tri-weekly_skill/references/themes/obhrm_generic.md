# 主题档：OBHRM 通用（运行时报主题，不必预写文件）
> 节点①交互使用：用户当场报"本期主题 + 板块清单"，本档据此动态成档。

## 主题闸门（运行时确定）
闸门 = 〔用户本期所报主题〕 × OBHRM（组织/管理/人/团队/制度/雇佣），且该条须能归入用户本期所给板块之一。
- **默认主题 = AI×OBHRM**（等同 `ai_obhrm.md`，用户不另说时走这个）。
- 其他主题示例：组织变革、组织文化、招聘甄选、薪酬、绩效、员工发展/培训、HR 三支柱(SSC)、领导力、团队动力学(Team Dynamics)、雇佣关系……
- 判断"是否属于本期主题"以**用户所报主题 + 板块**为准；纯无关一律排除。低置信度边缘条目交用户定夺（保召回 + 人审）。

## 板块（运行时确定）
用户在节点①手动给出本期板块清单（**数目不限**，生成器按 JSON 里的板块循环）。
- AI×OBHRM 默认沿用既有五板块（技术对工作/对组织/人的技能/安全隐私感受热点/最佳实践）。
- 未给清单时按主题给一套通用默认：理论/机制 → 实证发现 → 方法/测量 → 情境与边界条件 → 实践启示。

## 锚点（few-shot）
无预置示例时，按"标题+摘要是否落在用户所报主题、且匹配某板块"判断。某主题若常做、想要更高分类精度，可另写 `references/themes/<主题>.md` 补 few-shot（专门档分类略优）。

## 白名单
复用 `source_registry.md`（FT50 + ABS 六组，覆盖全 OBHRM），**不随主题改**。
